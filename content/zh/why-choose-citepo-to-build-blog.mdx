如果你是开发者或 AI 初创团队的成员，你一定深有体会：你需要一个博客来分享动态、建立品牌影响力、获取自然流量 —— 但搭建博客不应该像造一个新产品那么复杂。

这正是 CitePo 存在的理由。

## 现有方案的问题

### WordPress / Ghost

功能强大的 CMS 平台，但带来的负担也不小：

- **运维成本** —— 需要服务器、数据库，持续维护
- **安全隐患** —— 频繁的更新和漏洞修复
- **大材小用** —— 你只是想发布 Markdown，而不是管理一个 CMS

<Callout type="warning">
传统 CMS 需要持续的服务器维护和安全补丁——这些精力本应用来写内容。
</Callout>

### 静态站点生成器（Hugo、Jekyll、Gatsby）

更接近开发者想要的，但仍有摩擦：

- **复杂配置** —— 主题、插件、构建流水线
- **依赖地狱** —— 数百个包，频繁出问题
- **仓库 90% 都是引擎代码** —— 真正的内容被框架代码淹没

<Callout type="warning">
使用 SSG，你最终在维护一套复杂的构建系统，而不是专注于内容本身。
</Callout>

### Medium / Substack

容易上手，但：

- **没有所有权** —— 内容寄存在别人的平台上
- **品牌受限** —— 和平台上所有博客长得一样
- **SEO 受限** —— 你在为平台的域名做贡献，而不是自己的

<Callout type="warning">
托管平台用便利换控制权——你的内容、品牌和 SEO 都属于别人。
</Callout>

## 快速开始

创建博客只需几步：

<Steps>
  <Step title="创建博客">
    运行脚手架命令生成新博客项目：
    ```bash
    npx citepo new my-blog
    ```
  </Step>
  <Step title="进入项目目录">
    ```bash
    cd my-blog
    ```
  </Step>
  <Step title="启动开发服务器">
    本地预览你的博客：
    ```bash
    npx citepo dev
    ```
  </Step>
</Steps>

就这么简单。无需注册账号，无需信用卡，无需配置向导。直接开始写作。
