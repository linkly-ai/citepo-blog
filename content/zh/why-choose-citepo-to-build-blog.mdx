每个开发者都有过这样的经历：你想要一个博客，用来分享正在做的事情、建立个人或团队的影响力、为产品获取自然流量。但每次坐下来搭建博客的时候，你花了几个小时折腾工具链，一个字也没写出来。

这不是你的问题。这是整个开发者社区最常见的困扰之一，也是我们创建 CitePo 的原因。

## 博客的两难困境

当前的博客生态存在一个令人不舒服的事实：现有工具迫使你在便利性和控制权之间做出取舍。

Medium、Substack 这类平台让你几分钟内就能开始写作，但你的内容寄托在别人的域名上、别人的品牌下、受制于别人的规则。本质上你是在为平台积累 SEO 权重，而读者和你之间的关系也归平台所有。写个人日记或许无所谓，但对于正在建立专业形象的开发者或创业团队来说，这不可接受。

![开发者的工作台，笔记本电脑与笔记本](https://images.unsplash.com/photo-1517694712202-14dd9538aa97?w=960&q=80)

另一极是 WordPress 或 Ghost 这类自托管方案。它们给了你完全的控制权，但也带来了运维负担 —— 服务器要维护、数据库要备份、安全补丁要打。你只是想要一个博客，不是想当运维工程师。

Hugo、Jekyll、Gatsby 等静态站点生成器介于两者之间。理论上它们对开发者很友好，但实际操作中，你的仓库有 90% 都是框架代码，真正的内容只占 10%。配置文件、插件依赖、主题模板代码层层叠加。每次隔几个月回来，总有什么东西坏了，你得先花一下午更新依赖，才能写下第一个字。

## 一种不同的思路

CitePo 采用了一个根本性不同的立场：**你的博客仓库应该只包含你的内容**。

没有 `package.json`，没有 `node_modules`，没有主题文件占据你的仓库。当你创建一个 CitePo 博客时，Git 仓库里只有真正重要的东西 —— MDX 格式的文章、一个配置文件、你的图片素材。仅此而已。

引擎、主题、构建流水线 —— 这些全部由 CitePo 管理。当我们改进主题或添加新功能时，你无需动仓库里的任何东西就能自动获得更新。你的仓库始终保持干净、专注，即使一年后回来看也能立刻理解。

这种分离对写作体验有深远的影响。打开你的博客仓库，感觉就像打开一个文件夹，而不是一个软件项目。没有认知负担，没有构建系统要理解，没有框架要学习。

## Git 即真相之源

CitePo 的一切都通过 Git 流转。你用自己喜欢的编辑器写作 —— VS Code、Cursor、Vim，随你选择。用 `npx citepo dev` 本地预览。满意之后 `git push`，博客就部署了。

这套工作流的好处远不止简单。因为内容在 Git 里，你天然拥有版本历史。你可以用 Pull Request 来审核草稿，通过 Git 平台邀请协作者，AI 编程助手可以直接在编辑器里帮你写作和修改。不需要学习任何私有 CMS，也不会遇到网页编辑器在浏览器崩溃时丢失你作品的情况。

对团队来说这尤其强大。产品更新可以走和代码变更一样的 PR 审核流程，市场和工程团队在同一套工作流里协作，一切都可审计、可回滚、透明清晰。

## 为 AI 时代而生

我们带着一个信念构建 CitePo：在未来几年，你博客的很大一部分"读者"将是代替人类收集信息的 AI 助手。

因此，每个 CitePo 博客都会自动生成 `llms.txt` —— 一份机器可读的索引，帮助 AI 助手理解和引用你的内容。结合 JSON-LD 结构化数据，你的文章会成为 AI 系统知识图谱的一部分。这不是噱头，而是内容被发现和消费方式的根本性转变。

<Callout type="tip">
  CitePo 自动生成 `llms.txt`、JSON-LD 结构化数据、sitemap、RSS 和 Open Graph
  标签。你的内容从发布那刻起，就同时为人类读者和 AI 助手做好了优化。
</Callout>

## 无需费力，即刻精美

CitePo 附带精心设计的主题，无需任何定制就能呈现出色的视觉效果。排版针对中、英、日三语长文阅读做了专门优化，暗色模式无缝切换，布局响应式且干净，代码块自带语法高亮。一切开箱即用。

我们对细节的执着是为了让你不必操心。Lighthouse 评分稳定在 99 以上，页面瞬间加载 —— 因为 Astro 生成的是纯静态 HTML，JavaScript 体积极小。阅读体验快速、无干扰、无障碍。

## 快速开始

创建一个博客只需三条命令：

```bash
npx citepo new my-blog
cd my-blog
npx citepo dev
```

这就是全部的搭建过程。不需要注册账号，不需要信用卡，不需要配置向导。博客已经在本地运行，等待你的第一篇文章。准备好上线时，将仓库连接到 CitePo 托管，之后每次 `git push` 都会自动触发部署。
