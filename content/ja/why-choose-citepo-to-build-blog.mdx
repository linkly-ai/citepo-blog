すべての開発者が経験したことがあるはずです。ブログが欲しい — 自分が作っているものを共有し、影響力を築き、プロダクトへのオーガニックトラフィックを増やすために。でもいざブログを構築しようとすると、ツールチェーンとの格闘に何時間も費やして、一文字も書けないまま終わる。

これはあなただけの問題ではありません。開発者コミュニティで最もよくある悩みの一つであり、私たちが CitePo を作った理由です。

## ブログのジレンマ

現在のブログエコシステムには不都合な真実があります。既存のツールは、利便性とコントロールのどちらかを選ぶことを強いてくるのです。

Medium や Substack のようなプラットフォームなら数分で書き始められます。しかしあなたのコンテンツは他社のドメインに、他社のブランドの下に、他社のルールに従って存在することになります。本質的にはプラットフォームの SEO 権威を築いているのであり、読者との関係もプラットフォームのものです。個人的な日記なら構わないかもしれませんが、プロフェッショナルな存在感を築こうとしている開発者やスタートアップにとっては、受け入れられない条件です。

![ラップトップとノートのある開発者のデスク](https://images.unsplash.com/photo-1517694712202-14dd9538aa97?w=960&q=80)

対極にあるのが WordPress や Ghost のようなセルフホスト型ソリューションです。完全なコントロールが得られますが、運用の負担も伴います — サーバーのメンテナンス、データベースのバックアップ、セキュリティパッチの適用。ブログが欲しかっただけなのに、システム管理者の仕事を抱えることになります。

Hugo、Jekyll、Gatsby といった静的サイトジェネレーターはその中間に位置します。理論上は開発者フレンドリーですが、実際にはリポジトリの 90% がフレームワークコードで、実際のコンテンツは 10% しかありません。設定ファイル、プラグインの依存関係、テーマのボイラープレートが積み重なります。数ヶ月ぶりに戻ってくるたびに何かが壊れていて、一文字書く前に午後いっぱい依存関係のアップデートに費やすことになります。

## 異なるアプローチ

CitePo は根本的に異なる立場を取ります：**ブログのリポジトリにはコンテンツだけがあるべき**です。

`package.json` なし。`node_modules` なし。テーマファイルがリポジトリを散らかすこともなし。CitePo でブログを作成すると、Git リポジトリには本当に大切なものだけが含まれます — MDX 形式の記事、一つの設定ファイル、画像素材。それだけです。

エンジン、テーマ、ビルドパイプライン — これらはすべて CitePo が管理します。テーマの改善や新機能の追加があっても、リポジトリには一切触れずに自動的にアップデートが反映されます。リポジトリは常にクリーンで集中的な状態を保ち、1年後に戻ってきてもすぐに理解できます。

この分離は執筆体験に深い影響を与えます。ブログのリポジトリを開くと、ソフトウェアプロジェクトではなくドキュメントフォルダを開いている感覚になります。認知的な負荷がなく、理解すべきビルドシステムもなく、学ぶべきフレームワークもありません。

## Git が信頼の源泉

CitePo のすべては Git を通じて流れます。好きなエディタで執筆します — VS Code、Cursor、Vim、お好みで。`npx citepo dev` でローカルプレビュー。満足したら `git push` でブログがデプロイされます。

このワークフローの利点はシンプルさだけではありません。コンテンツが Git にあるので、バージョン履歴が自然に手に入ります。Pull Request でドラフトをレビューでき、Git プラットフォームを通じてコラボレーターを招待できます。AI コーディングアシスタントがエディタ内で直接執筆や編集を手伝ってくれます。独自の CMS を学ぶ必要もなく、ブラウザがクラッシュした時に作品を失う Web エディタの心配もありません。

チームにとってこれは特に強力です。プロダクトのアップデートはコード変更と同じ PR レビュープロセスを経ることができます。マーケティングとエンジニアリングが同じワークフローで協力し、すべてが監査可能で、巻き戻し可能で、透明です。

## AI 時代のために構築

私たちは一つの確信を持って CitePo を構築しました：今後数年で、ブログの「読者」のかなりの部分は、人間に代わって情報を収集する AI アシスタントになるだろうと。

そのため、すべての CitePo ブログは自動的に `llms.txt` を生成します — AI アシスタントがコンテンツを理解し参照するための機械可読なインデックスです。JSON-LD 構造化データと組み合わせることで、あなたの記事は AI システムが参照するナレッジグラフの一部になります。これはギミックではなく、コンテンツが発見され消費される方法の根本的な変化です。

<Callout type="tip">
  CitePo は `llms.txt`、JSON-LD 構造化データ、sitemap、RSS、Open Graph
  タグを自動生成します。公開した瞬間から、人間の読者と AI アシスタントの両方に最適化されています。
</Callout>

## 手間なく、すぐに美しく

CitePo にはカスタマイズなしでも素晴らしい見栄えの、丁寧にデザインされたテーマが付属しています。タイポグラフィは英語、中国語、日本語の長文読書に最適化されており、ダークモードはシームレスに切り替わり、レイアウトはレスポンシブでクリーン、コードブロックはシンタックスハイライト付き。すべてがそのまま動きます。

私たちがディテールにこだわるのは、あなたがそうしなくて済むようにするためです。Lighthouse スコアは常に 99 以上。Astro が最小限の JavaScript で純粋な静的 HTML を生成するため、ページは瞬時に読み込まれます。読書体験は高速で、邪魔がなく、アクセシブルです。

## はじめよう

ブログの作成はたった3つのコマンドで完了します：

```bash
npx citepo new my-blog
cd my-blog
npx citepo dev
```

セットアップはこれだけです。アカウント登録不要、クレジットカード不要、設定ウィザード不要。ブログはローカルで稼働しており、最初の記事を待っています。公開の準備ができたら、リポジトリを CitePo のホスティングに接続すれば、`git push` のたびに自動デプロイが走ります。

## これからの展望

CitePo は積極的に進化を続けています。カスタムドメイン、プライバシー重視のアナリティクス、追加テーマ、マルチオーサーサポートがロードマップに載っています。私たちはずっと欲しかったブログプラットフォーム — あなたの時間、コンテンツ、所有権を尊重するプラットフォーム — を構築しています。

Git リポジトリのようにシンプルで、プロがデザインしたサイトのように洗練されたブログを望んだことがあるなら、ぜひ CitePo を試してみてください。すぐに違いを感じていただけるはずです。
