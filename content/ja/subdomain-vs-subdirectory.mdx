ウェブサイトにブログを追加する際、一見シンプルな疑問が浮かびます：ブログは `blog.example.com`（サブドメイン）に置くべきか、それとも `example.com/blog`（サブディレクトリ）に置くべきか？

この決定は、サイトのSEOパフォーマンス、管理の複雑さ、そして長期的なコンテンツ戦略に影響を与えます。この2つのアプローチの違いと、正しい選択をする方法について詳しく見ていきましょう。

## 技術的定義

まず、この2つの概念を明確にしましょう：

**サブドメイン（Subdomain）**：メインドメインの前にプレフィックスを追加し、独立した第三レベルドメインを形成します。

```
blog.example.com
shop.example.com
docs.example.com
```

**サブディレクトリ（Subdirectory）**：メインドメインの後にパスを追加し、同じドメイン下でコンテンツをホストします。

```
example.com/blog
example.com/shop
example.com/docs
```

![サブドメインとサブディレクトリのURL構造比較](images/subdomain-vs-subdirectory.webp)

技術アーキテクチャの観点から、サブドメインは完全に異なるサーバーを指すことができますが、サブディレクトリは通常同じサーバー上に存在します。この違いは、検索エンジンがコンテンツをどのように理解し処理するかに大きく影響します。

## Googleの公式見解

Google Search AdvocateのJohn Mueller氏は、何度も公に発言しています：

> "In general, we see these the same. I would personally try to keep things together as much as possible."
>
> （一般的に、私たちはこれらを同じように見ています。個人的には、できるだけコンテンツをまとめておくことをお勧めします。）

別の回答では、さらに詳しく説明しています：

> "Google Web Search is fine with using either subdomains or subdirectories... use what works best for your setup and think about your longer-term plans when picking one or the other."
>
> （Google Web検索は、サブドメインでもサブディレクトリでも問題ありません...あなたの設定に最適なものを使用し、どちらかを選ぶ際には長期的な計画を考慮してください。）

Googleの公式見解は明確です：**技術的な観点から、検索アルゴリズムは両方の構造を同等に処理できます**。しかし注目すべきは、Googleの公式代表者でさえ、個人的にはサブディレクトリを推奨する傾向があるということです。

## 業界専門家の異なる見解

Googleが両者は同等だと主張しているにもかかわらず、SEOコミュニティの実践経験は異なる物語を語っています。

著名なSEO専門家たちがそれぞれの見解を共有しています：

- **Gianluca Fiorelli**（MOZ/I Love SEO）：サブディレクトリが最良の選択、権威が集中するため
- **Stephen Kenwright**（Branded3）：サブディレクトリはほぼ常にサブドメインより優れている
- **Eric Enge**（Stone Temple）：新規サイトにはサブディレクトリを推奨、SEOのためだけに既存のサブドメインを移行する価値はない

### 実際のケーススタディ

以下は公開されている移行事例です：

| ケース        | 変更                            | 結果                          |
| ------------- | ------------------------------- | ----------------------------- |
| Chubo Knives  | サブドメイン → サブディレクトリ | オーガニックトラフィック+421% |
| Monster.co.uk | サブドメイン → サブディレクトリ | 検索可視性+116%               |
| HotPads       | blog.hotpads.com → /blog/       | トラフィック2倍               |
| Flick         | サブドメイン → サブディレクトリ | 12ヶ月で年間960万トラフィック |

もちろん、失敗したケースもあります。IWantMyNameはかつてサブディレクトリからサブドメインに移行しましたが、トラフィックが大幅に減少し、元に戻さざるを得ませんでした。

<Callout type="warning">
  注意すべき点として、これらの成功事例は多くの場合、プラットフォームの再構築、内部リンクの改善、技術的問題の修正など、他の最適化施策を伴っています。構造の移行だけでは、同じ結果が保証されるわけではありません。
</Callout>

## 技術的分析

なぜ実際にはサブディレクトリの方が良いパフォーマンスを示す傾向があるのでしょうか？技術的な観点から分析してみましょう。

### ドメインオーソリティの継承

サブディレクトリは、メインドメインのすべての権威を自然に継承します。誰かが `example.com/blog/article` にリンクすると、その被リンクは `example.com` ドメイン全体を直接強化します。

一方、サブドメインは検索エンジンによって半独立のエンティティとして扱われます。`blog.example.com` へのリンクは主にサブドメイン自体に価値が蓄積され、メインドメインへの貢献はより間接的になります。

### 内部リンクの価値

- **サブディレクトリ**：すべての内部リンクが真の内部リンクとなり、権威の継承がスムーズ
- **サブドメイン**：サブドメイン間のリンクは「外部リンク」として扱われる可能性があり、権威の継承が希薄化

### クローラーの動作の違い

John Mueller氏は、Googleクローラーの動作の違いについて説明しています：

> サブディレクトリは、すべてのコンテンツが同じサーバー上にあることがわかっているため、クロールに役立ちます。同様にクロールできます。
>
> サブドメインも機能します...ただし、それらを個別にクロールすることを学ぶ必要があり、これは最初の数日間の形式的なものです。

「数日間」の問題に過ぎませんが、新規サイトや新しいコンテンツにとって、この学習期間はインデックス速度に影響を与える可能性があります。

### 管理の利便性

サブディレクトリを使用する場合、必要なのは：

- 1つのSSL証明書
- 1つの分析設定（Google Analytics / Search Console）
- 1つのホスティング環境

サブドメインの場合、これらをそれぞれ個別に設定する必要があり、運用の負担が増加します。

## どう選ぶべきか？

### サブディレクトリが推奨されるシナリオ

| シナリオ                        | 理由                                        |
| ------------------------------- | ------------------------------------------- |
| ブログ                          | メインサイトの権威を共有、全体的なSEOを向上 |
| 製品ページ                      | 統一されたブランディング、管理の簡素化      |
| リソースセンター/ナレッジベース | 内部リンクの効果を最大化                    |
| 新規サイト                      | 権威を集中させる方が効率的                  |

### サブドメインが適切なシナリオ

| シナリオ                 | 理由                                 |
| ------------------------ | ------------------------------------ |
| 国際化サイト             | es.example.com, fr.example.com       |
| 独立したアプリ/ツール    | app.example.com は個別デプロイが必要 |
| 完全に異なる事業ライン   | ブランドの分離が必要                 |
| 開発/テスト環境          | staging.example.com                  |
| 技術アーキテクチャの制約 | メインドメイン下でホストできない     |

<Callout type="tip">
  オーガニックトラフィックを獲得するためにウェブサイトにブログを追加したいだけなら、サブディレクトリがほぼ常に良い選択です。
</Callout>

## CitePo の推奨

[SEOとGEOの両方に最適化されたブログプラットフォーム](/ja/why-choose-citepo-to-build-blog)として、CitePo は両方のデプロイ方法をサポートしています：

**すでにウェブサイトをお持ちの場合**、サブディレクトリでのデプロイ（例：`example.com/blog`）を強くお勧めします。ブログコンテンツがメインドメインの権威を直接構築します—これがコンテンツマーケティングの核心的な価値です。

**まったく新しい独立したブログの場合**、サブドメインも合理的な選択です。この場合、`blog.yourname.com` のような構造を使用できます。

CitePo のドメイン設定では、この2つの方法を自由に選択でき、最もコストパフォーマンスの高い価格設定となっています：

> - ドメインの追加は無料
> - Pro プランを購読するだけでサブディレクトリを追加可能—他のブログプラットフォームのように追加料金を支払ったり、エンタープライズプランを購読する必要はありません

セットアップは数分で完了し、どちらの方法を選んでも同じ高品質なホスティングサービスを受けられます。

## まとめ

重要なポイントを振り返りましょう：

1. **Googleの公式見解**では、両者は技術的に同等と主張
2. **SEOの実践**では、一般的にサブディレクトリの方がパフォーマンスが良い
3. **技術的な理由**には、権威の継承、内部リンクの効果、管理の利便性が含まれる
4. **ほとんどのシナリオ**では、サブディレクトリがより安全な選択
5. **特定のシナリオ**（国際化、独立したアプリ）では、サブドメインがより適切

ウェブサイトにブログを追加する場合、サブディレクトリが私たちの推奨するデフォルトの選択です。これはSEOのベストプラクティスであるだけでなく、最もシンプルな技術的ソリューションでもあります。
